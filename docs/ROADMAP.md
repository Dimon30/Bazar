# Roadmap: Bazar ML Marketplace Service

## Цель проекта

Bazar — учебно-практический ML-сервис маркетплейса, который объединяет:

- поиск товаров;
- ранжирование выдачи;
- динамическое ценообразование;
- рекомендации;
- сбор пользовательских событий;
- версионирование моделей;
- API для будущего интерфейса.

Главная цель — собрать не просто ML-модель, а end-to-end ML-систему: от данных и обучения до API, инференса и деплоя.

## Не делать на раннем этапе

- Не начинать с PySpark, если данных мало.
- Не начинать с Kubernetes.
- Не тащить PyTorch до появления стабильного baseline.
- Не делать сложный UI до рабочего backend.
- Не подключать MLflow раньше простого model registry.
- Не переносить legacy целиком.

---

## Главный end-to-end сценарий

```text
Пользователь вводит поисковый запрос
        ↓
FastAPI получает запрос
        ↓
Backend достаёт кандидатов из базы
        ↓
RankingService сортирует товары
        ↓
PricingService считает финальную цену
        ↓
API возвращает top-K товаров
        ↓
UI показывает результат
        ↓
События пользователя сохраняются в events
        ↓
Новые события используются для улучшения моделей
```
